Overview

Decision Matrix is designed with your privacy as the top priority. This privacy policy explains how we handle (or rather, don't handle) your data.

ðŸŽ¯ The Short Version

We don't collect, store, or transmit any of your personal data. Period.

Everything happens in your browser. Your decisions, your criteria, and your ratings never leave your device unless you explicitly choose to share them.

No Data Collection

Unlike most web applications, Decision Matrix operates entirely in your web browser with zero data collection:

  • No server-side storage: We don't have a backend database or servers collecting your data
  • No analytics or tracking: We don't use Google Analytics, Facebook Pixel, or any other tracking services
  • No cookies: We don't set any cookies to track your browsing behavior
  • No third-party services: We don't integrate with any external APIs or services
  • No user accounts: We don't require sign-ups or collect email addresses
  • No IP logging: We don't log your IP address or any identifying information

How Your Data is Stored

Decision Matrix uses two privacy-friendly methods to store your data, both completely local to your device:

1. URL-Based Storage (Primary Method)

Your decision matrix data is encoded directly into the page URL using Base64 encoding:

How it works:

// Your data structure
const data = {
    options: [...],
    criteria: [...],
    ratings: {...},
    weights: {...}
};

// Encoded in the URL hash
const encoded = btoa(encodeURIComponent(JSON.stringify(data)));
window.location.hash = encoded;

Benefits:

  • Easy sharing - just copy and paste the URL
  • No server required - works completely offline
  • Transparent - you can see your data in the URL
  • Bookmarkable - save your decisions in browser bookmarks

2. Browser Local Storage (Optional)

When you click "Save to Browser", your matrix is stored in your browser's localStorage:

Implementation:

// Save to localStorage
const storageKey = 'decisionMatrix_data';
const data = { options, criteria, ratings, weights };
localStorage.setItem(storageKey, JSON.stringify(data));

// Load from localStorage
const stored = localStorage.getItem(storageKey);
const data = JSON.parse(stored);

Important notes about localStorage:

  • Data stays on your device only
  • Not accessible to other websites
  • Persists across browser sessions
  • You can clear it anytime in browser settings
  • Typically 5-10 MB storage limit per domain

Theme Preference Storage

The only other piece of data we store locally is your theme preference (light/dark mode):

// Storing theme preference
localStorage.setItem('theme', 'dark'); // or 'light'

// This is the ONLY other localStorage item we use

No External Connections

Our Content Security Policy (CSP) ensures that the application cannot make any external network requests:

Content-Security-Policy:
    default-src 'self';
    script-src 'self';
    style-src 'self';
    img-src 'self' data:;
    connect-src 'self';

This means:

  • All JavaScript files are loaded from our domain only
  • All CSS files are loaded from our domain only
  • No external fonts, images, or resources
  • No AJAX requests to third-party APIs
  • Your browser actively blocks any unauthorized connections

Data Sharing

You control all data sharing:

Share Link Feature

When you click "Copy Share Link", the application copies the current page URL (with your data encoded in it) to your clipboard. This is the only way your data can be shared:

  • Voluntary action by you
  • You choose who to share the link with
  • Anyone with the link can view your matrix
  • No server involved in the sharing process

Export Features

You can export your matrix as JSON, CSV, or text files:

// Export creates a download - no upload anywhere
const blob = new Blob([content], { type: mimeType });
const url = URL.createObjectURL(blob);
// File downloads to your device only

Security Measures

We implement multiple security measures to protect your privacy:

  • Content Security Policy: Prevents unauthorized script execution and data exfiltration
  • No inline scripts: All code is in external, auditable files
  • HTTPS only: All connections are encrypted
  • No external dependencies: No third-party JavaScript libraries that could track you
  • Input validation: All user inputs are validated to prevent injection attacks

GDPR & Privacy Regulations

Because we don't collect any personal data, most privacy regulations don't apply to this application:

  • CCPA (California): No personal information sold or shared

  • Data portability: Export your data anytime with our export features
